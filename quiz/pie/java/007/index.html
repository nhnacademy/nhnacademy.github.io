interface Top {
  default void accept(Visitor v) {
    v.visit( this );
  }
}

class Left implements Top {
}

class Right implements Top {
}

interface Visitor {
  default void visit(Top top) { throw new NoSuchMethodError(); }
  void visit(Left left);
  void visit(Right right);
}

class 걸어 implements Visitor {
  public void visit(Left left) { System.out.println("왼쪽으로 걸어");};
  public void visit(Right right) { System.out.println("오른쪽으로 걸어");}
}

class 뛰어 implements Visitor {
  public void visit(Left left) { System.out.println("왼쪽으로 뛰어");};
  public void visit(Right right) { System.out.println("오른쪽으로 뛰어");}
}

class 실험 {
  public static void main(String[] args) {
    Top leftTop = new Left();
    Top rightTop = new Right();
    leftTop.accept( new 걸어() );
    rightTop.accept( new  뛰어() );
  }
}

도움말

귀띔

  • Java의 method overriding란 dynamic dispatch (on type), 다시 말해, 아래 보기에서처럼 evaluate 인자 e의 runtime type에 따라 그에 대응하는 eval 연산을 불러주는 코드를 간추린 표현입니다.
interface E {}
class N implements E {}
class V implements E {}
abstract class B implements E {}
class A extends B {}
class M extends B {}
class Evaluator {
	public static E evaluate(E e, Env rho) {
		if (e instanceof N) evaluateN(e, r)
		else if (e instanceof V) evaluateV(e, r)
		else if (e instanceof A) evaluateA(e, r)
		else if (e instanceof M) evaluateM(e, r)
		else
				// error : unknown subtype instance of E
    }
}
  • 데이터 타입을 표현한 코드에는 거의 손대지 않고, 새로운 연산이 늘어나는 경우에는 Evaluator 처럼 연산을 다른 모듈로 솎아내어 데이터 타입과 연산을 분리하는 설계 방식 곧 Visitor 패턴을 쓰는 것이 더 나을 때가 많습니다. 이와 함께 single dispatch를 이해해야 풀 수 있는 문제입니다.
